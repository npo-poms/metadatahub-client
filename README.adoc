= MetadataHub SPARQL Client


https://docs.metadatahub.bijnpo.nl/handleiding/developers/sparql-service/

== Description



link:src/main/java/nl/npo/metadatahub/client/MetadatahubClient.java[MetadatahubClient]:: A client to wrap some sparql and editorial endpoints of the metadatahub.
- methods to execute sparql queries
- method to retrieve as JSON objects from editorial api



link:src/main/java/nl/npo/metadatahub/poms/PomsService.java[PomsService]:: A service wrapping MetadatahubClient to retrieve date from metadatahub and map it to POMS objects. link:src/main/resources/sparql/[Used sparql queries]

link:src/test/java/QuerySingleProgram.java[QuerySingleProgram.java]:: this is an example of how to use the PomsService to retrieve specific objects from the metadatahub and map it to POMS objects. It writes some resulting files to link:results/single[results].

link:src/test/java/QueryDays.java[QueryDay.java]:: This finds all programs for number of days/channel combination. It writes some resulting files to link:results[results] too, which can be used as a example data of how it looked in poms, and how it looks after migration to the new metadata hub, and mapping back to poms (to make it easier comparable)

== How to use

The two 'compact source' files are in `src/test/java` which makes it easy to just run them from your IDE.

A jar containing the classes from `src/main/java` is available via maven central (for now a snapshot)

For now, this is just a demo for reference. It simply uses the java LTS 25, and jackson3 and so on. . If someone wants to actually use it, and is stuck with a lower version (for some reason) of java or jackson or so, I'll gladly make it compatible with your version (if, at least, java 11). Just let me know.
